Welcome

If you go to this page, this probably means that you are interested in contributing to Kloxo by coding and we certainly welcome you. Kloxo is an open-source program which means that evey body have the right to edit the code. In particular, we use a method called Pull Requests to handle codes contributed by the general public.

To start...

To start, you should first have your own GitHub developer account. Then, click on the little Fork botton. Right now, a dedicated kloxo fork is created in your account and you're ready to go! LxCenter uses Redmine to track bugs. Therefore, the next step would be going to the bug tracker. Then, click on Kloxo, then Issues. You will be able to see a list of bug, feature requests etc. Choose one that you are interested in and is not assigned to anybody. Leave a message that you are right now working on it and officially start your development!

In your own kloxo fork, create a branch called {type}-{issue id}, for instance, bug-1000. Right now, you can actually start coding for the issue with any IDE or text editor you like!

Coding Standard

Every open-source project have its rules for developers and so do LxCenter. Here, it is called Coding Standard. Every developer must follow the standard or your code might not be accepted.

Serveral important rules are: 1) commit on one single thing at one time; 2) follow the GNU AGPL requirement and; 3) follow general PHP coding standard. The detailed Coding Standard is available on the project site.

Do also noted that we have specific rules for the commit messages so they are also important.

When the coding is done...

When the coding is done, you should update us at the bug tracker, then submit a Pull Request. Go back to Kloxo's GitHub repository, then click the little Pull Request button on the top right hand corner. In base repo, choose lxcenter/kloxo and in base branch, choose dev. In head repo, choose the name of your own fork and in head branch, choose the branch you created for the issue.

In the title of the Pull Request, you are suggested to put down the title of the issue and the ID of the issue which is availabe on the bug tracker.

Core review

In LxCenter, there is a group of experienced developer called Cores and a memeber is called Core. After submitting the Pull Request, a Core will review your request. Then, they will either approve your request, or else give you comments on how to improve your request. In rare circumstances however, a request may be rejected.

Benefits for contributing

If you have made more than 3 approved Pull Request per three months, you may be entitled to become a LxCenter Representative. A Representative is entitled to be referred with paid support requests LxCenter receives and can made comments on important LxCenter changes. Developers who have confidence that you have made valuable contribution to LxCenter can also apply to the Cores. For more details, please email us at contact@lxcenter.org.

For a helping hand...

We have a public developer mailinglist hosted by Google at lxdevel@googlegroups.com. You may also search the group at Google Groups with the name LxCenter Development.
